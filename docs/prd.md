# 需求文档

## 1. 应用概述

### 1.1 应用名称
仿微信聊天系统

### 1.2 应用描述
一款即时通讯Web应用，提供用户注册登录、联系人管理、私聊、群聊、消息实时推送、会话列表、个人资料管理等核心功能，整体UI风格仿微信绿色主题。

## 2. 用户与使用场景

### 2.1 目标用户
需要进行即时通讯的个人用户或团队成员。

### 2.2 核心使用场景
- 用户通过注册账号后登录系统
- 添加好友并进行一对一私聊
- 创建群组进行多人群聊
- 实时接收和发送消息
- 查看会话列表和未读消息
- 管理个人资料信息

## 3. 页面结构与功能说明

### 3.1 页面结构

```
├── 注册页面
├── 登录页面
├── 主界面
│   ├── 会话列表页
│   ├── 联系人页
│   │   ├── 好友列表
│   │   └── 添加好友
│   ├── 聊天页面
│   │   ├── 私聊页面
│   │   └── 群聊页面
│   └── 个人中心页
│       └── 个人资料编辑页
```

### 3.2 功能说明

#### 3.2.1 注册页面
- 用户输入用户名、密码完成注册
- 用户名唯一性校验
- 注册成功后跳转至登录页面

#### 3.2.2 登录页面
- 用户输入用户名、密码进行登录
- 登录成功后进入主界面会话列表页

#### 3.2.3 会话列表页
- 展示最近对话的会话列表
- 每个会话显示：对方头像、昵称、最后一条消息内容、时间、未读消息数
- 点击会话进入对应的聊天页面
- 未读消息数以红色数字角标显示

#### 3.2.4 联系人页

**好友列表**
- 展示已添加的好友列表
- 每个好友显示：头像、昵称
- 点击好友进入私聊页面

**添加好友**
- 通过用户名搜索用户
- 搜索结果展示用户头像、昵称
- 点击添加按钮发送好友请求
- 对方同意后成为好友

#### 3.2.5 私聊页面
- 顶部显示对方昵称
- 消息区域展示聊天记录
- 每条消息显示：发送者头像、消息内容、发送时间、已读/未读状态
- 底部输入区域支持：
  - 文字消息输入与发送
  - 图片选择与发送
  - 表情选择与发送
- 消息实时推送显示

#### 3.2.6 群聊页面
- 顶部显示群组名称
- 消息区域展示群聊记录
- 每条消息显示：发送者头像、昵称、消息内容、发送时间
- 底部输入区域支持：
  - 文字消息输入与发送
  - 图片选择与发送
  - 表情选择与发送
- 群成员管理入口（查看成员列表、添加成员、移除成员）
- 创建群组功能：选择好友创建新群组
- 消息实时推送显示

#### 3.2.7 个人中心页
- 展示当前用户头像、昵称、用户名
- 提供个人资料编辑入口

**个人资料编辑页**
- 支持修改头像（上传图片）
- 支持修改昵称
- 支持编辑个人信息
- 保存修改后更新个人资料

## 4. 业务规则与逻辑

### 4.1 用户注册与登录
- 用户名在系统内唯一，注册时需校验
- 密码需加密存储
- 登录成功后生成会话凭证

### 4.2 好友关系
- 添加好友需对方同意后建立双向好友关系
- 好友关系建立后双方可互相发送消息

### 4.3 消息发送与接收
- 私聊消息仅发送方和接收方可见
- 群聊消息所有群成员可见
- 消息通过WebSocket/Realtime实时推送
- 发送方发送消息后立即显示在聊天页面
- 接收方在线时实时接收消息，离线时下次登录后拉取未读消息

### 4.4 消息已读/未读状态
- 私聊消息：接收方打开聊天页面后标记为已读，发送方可看到已读状态
- 群聊消息：不展示已读状态
- 会话列表中显示未读消息数

### 4.5 会话列表排序
- 按最后一条消息时间倒序排列
- 有新消息时该会话自动置顶

### 4.6 群组管理
- 创建群组时创建者为群主
- 群主可添加成员、移除成员
- 群成员可查看群成员列表
- 群成员可主动退出群组

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 注册时用户名已存在 | 提示「用户名已被使用，请更换」 |
| 登录时用户名或密码错误 | 提示「用户名或密码错误」 |
| 搜索用户时未找到结果 | 提示「未找到该用户」 |
| 添加已是好友的用户 | 提示「对方已是你的好友」 |
| 发送消息失败 | 提示「消息发送失败，请重试」 |
| 网络断开连接 | 提示「网络连接已断开」，消息暂存本地待重连后发送 |
| 上传图片失败 | 提示「图片上传失败，请重试」 |
| 群组成员已达上限 | 提示「群组成员已达上限，无法添加」 |
| 非群主尝试移除成员 | 提示「仅群主可移除成员」 |

## 6. 验收标准

1. 用户在注册页面输入用户名、密码完成注册
2. 用户在登录页面输入用户名、密码成功登录
3. 用户在联系人页通过用户名搜索并添加好友
4. 用户在会话列表页点击好友进入私聊页面
5. 用户在私聊页面发送文字消息、图片、表情，对方实时接收
6. 用户在联系人页创建群组并添加好友
7. 用户在群聊页面发送消息，群成员实时接收
8. 用户在会话列表页查看未读消息数
9. 用户在个人中心页编辑头像和昵称
